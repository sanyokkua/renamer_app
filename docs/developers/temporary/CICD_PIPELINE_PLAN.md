# CI/CD Pipeline Overhaul — Implementation Plan

## Overview

| | Before | After |
|--|--------|-------|
| Workflows | 1 monolithic (`build-and-release.yml`) | 3 purpose-built |
| Feature branch CI | None | `ci.yml` — compile + test on Ubuntu |
| Main branch build | None | `build.yml` — build + package on all platforms |
| Release | Tag `*-release-*` branch pattern | `release.yml` — `v*` tag, version from tag |
| Linux arm64 | Missing | Added (`ubuntu-24.04-arm`) |
| Windows arm64 | N/A | No stable runner — excluded |

---

## Three Pipelines

### Pipeline 1: Feature Branch CI (`.github/workflows/ci.yml`)

**Trigger:** Push to any non-`main` branch, or PR targeting `main`.

**Purpose:** Fast feedback — does it compile, do tests pass?

**Runner:** `ubuntu-latest` (single)

**Steps:** Checkout → JDK 25 Temurin (cached) → `mvn verify`

No packaging, no artifacts. Build time: ~2-3 min.

---

### Pipeline 2: Main Branch Build (`.github/workflows/build.yml`)

**Trigger:** Push to `main` (tags excluded — release.yml handles those).

**Purpose:** Prove every commit to main is packageable on all platforms. 14-day artifacts for ad-hoc testing.

**Build job:** Ubuntu / Temurin → `mvn clean package` → uploads `app/ui/target/libs/` as `app-libs` artifact.

**Packaging matrix (parallel, `fail-fast: false`):**

| Job | Runner | JDK | Output |
|-----|--------|-----|--------|
| `package-macos` × 2 | `macos-13`, `macos-14` | Temurin | `.dmg` + `.tar.gz` |
| `package-windows` | `windows-latest` | Liberica Full (`jdk+fx`) | `.zip` (app-image) |
| `package-linux` × 2 | `ubuntu-latest`, `ubuntu-24.04-arm` | Temurin / Liberica Full | `.deb` + `.tar.gz` |

**APP_VERSION:** Placeholder `2.0.0` — snapshot artifacts, not releases.

---

### Pipeline 3: Tag Release (`.github/workflows/release.yml`)

**Trigger:** Tag matching `v[0-9]+.[0-9]+.[0-9]+` or `v[0-9]+.[0-9]+.[0-9]+-*`.

**Purpose:** Versioned release — inject version from tag, build, package all platforms, publish GitHub Release.

**Job flow:**

```
verify → build-and-test → package-macos (×2)    ─┐
                        → package-windows          ├→ create-release
                        → package-linux (×2)      ─┘
```

**Version extraction (in `verify` job):**

```bash
TAG="${GITHUB_REF_NAME}"              # e.g. v2.1.0-rc1
FULL_VERSION="${TAG#v}"               # → 2.1.0-rc1  (used in GitHub Release name)
JPACKAGE_VERSION="${FULL_VERSION%%-*}" # → 2.1.0     (used in Maven + jpackage)
```

- `v2.1.0` → jpackage `2.1.0`, stable release
- `v2.1.0-rc1` → jpackage `2.1.0`, GitHub pre-release (detected by `-` in tag name)

**Version injection:** `mvn versions:set -DnewVersion=2.1.0 -DgenerateBackupFiles=false` (CI workspace only — never committed).

**Guard:** `verify` job checks `git merge-base --is-ancestor <sha> origin/main` — tags not on main are rejected.

## Platform Matrix

| OS | Arch | Runner | JDK | Notes |
|----|------|--------|-----|-------|
| macOS | x86_64 | `macos-13` | Temurin | JavaFX dylibs embedded in Maven JARs |
| macOS | arm64 | `macos-14` | Temurin | Same — no Liberica Full needed |
| Linux | x86_64 | `ubuntu-latest` | Temurin | `fakeroot` installed for `.deb` |
| Linux | arm64 | `ubuntu-24.04-arm` | Liberica Full (`jdk+fx`) | `fakeroot` installed for `.deb` |
| Windows | x86_64 | `windows-latest` | Liberica Full (`jdk+fx`) | App-image only, no MSI |
| Windows | arm64 | — | — | No stable GitHub-hosted runner |

**Why Liberica Full on Windows and Linux arm64:** jpackage needs JavaFX `.jmod` files to link a self-contained JRE on these platforms. Liberica Full edition includes them. macOS and Linux x86_64 work with Temurin because jpackage resolves the JavaFX module graph from the modular JARs in `app/ui/target/libs/` (no `.jmod` files needed).

## Release Artifact Inventory

| File | Platform | Type |
|------|----------|------|
| `Renamer-{v}-macos-x86_64.dmg` | macOS Intel | Installer |
| `Renamer-{v}-macos-x86_64.tar.gz` | macOS Intel | Portable |
| `Renamer-{v}-macos-aarch64.dmg` | macOS Apple Silicon | Installer |
| `Renamer-{v}-macos-aarch64.tar.gz` | macOS Apple Silicon | Portable |
| `Renamer-{v}-windows-x86_64.zip` | Windows | Portable |
| `Renamer-{v}-linux-x86_64.tar.gz` | Linux x86_64 | Portable |
| `renamer_{v}_amd64.deb` | Linux x86_64 | Installer |
| `Renamer-{v}-linux-aarch64.tar.gz` | Linux arm64 | Portable |
| `renamer_{v}_arm64.deb` | Linux arm64 | Installer |

`.deb` names are generated by jpackage from the host arch — already arch-specific, no renaming needed.
macOS DMGs are renamed inline in the YAML (`Renamer-{v}.dmg` → `Renamer-{v}-macos-{arch}.dmg`) to prevent filename collision when both arch artifacts are collected.

## GitHub Secrets Required

None. `GITHUB_TOKEN` is provided automatically and has `contents: write` permission (declared in `release.yml`).

No code signing — app is distributed unsigned. Gatekeeper warning on macOS is documented in `docs/mac-os-requirements.md`.

## Developer Workflow

### Day-to-day

```
push feature/* → ci.yml (compile + test, ~2-3 min)
merge to main  → build.yml (full matrix package, ~15-20 min, artifacts available)
```

### Creating a release

```bash
git checkout main && git pull
git tag v2.1.0
git push origin v2.1.0
# release.yml triggers automatically
```

### Migration

1. Create all three new workflow files (Tasks 1–3).
2. Verify each has at least one successful run.
3. Delete `.github/workflows/build-and-release.yml` (Task 4).
